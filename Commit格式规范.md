# Commit格式规范

Commit由类型+内容+问题组成

``[<type>][message][issues]``

1. 类型 type

   * fix: bug 修复
   * feature: 新增功能
   * style: 代码风格调整
   * typo: 代码或文档勘误
   * refactor: 代码重构
   * test: 单元测试
   * chore: 构建工具的修改
   * reevert: 回滚
   * docs: 文档
2. 内容 message

   * 内容简单明了，可是中文也可是英文
3. 问题 issues

   * 有issues填issues，没issues填0


举例：

```
[docs][添加Commit格式规范][#0]
[feature][新增XX功能][#0]
[fix][修复XXBug][#46]
```
